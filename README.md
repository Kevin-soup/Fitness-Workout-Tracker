# Assignment 9 – Full Stack MERN App

## Overview
In this project, you will build a **full-stack MERN application** that tracks exercises completed by a user.  
The app has two main parts:
- A **REST API backend** using Node, Express, and MongoDB.  
- A **React frontend** SPA that interacts with the backend via HTTP requests.  

This is the **Portfolio Assignment** of the course, meaning you are allowed to post the entire project publicly (e.g., GitHub, portfolio site) after the term ends.

## Learning Goals
- Understand the lifecycle of a React component.  
- Use the `useEffect` and `useNavigate` hooks in React.  
- Connect a React frontend to a REST API backend across different servers.  
- Use the **Fetch API** for HTTP requests.  
- Manage component state and lift state up when needed.  

## Data Model
Each **exercise** document stored in MongoDB must have the following properties:

Property | Type   | Notes
---------|--------|-------------------------------------
name     | String | Must not be empty
reps     | Number | Integer > 0
weight   | Number | Integer > 0
unit     | String | Must be "kgs" or "lbs"
date     | String | Format: MM-DD-YY (e.g., 07-30-21)
_id      | Object | Auto-generated by MongoDB

## REST API (Backend)

### 1. Create Order – POST /exercises
Request Body:
```json
{ "name": "Push Ups", "reps": 20, "weight": 0, "unit": "kgs", "date": "07-30-21" }
```
Success (201): returns the created object with _id.  
Failure (400):
```json
{ "Error": "Invalid request" }
```

### 2. Read All – GET /exercises
Returns an array of exercise objects.

### 3. Read One – GET /exercises/:id
Success (200): returns the exercise object.  
Failure (404):
```json
{ "Error": "Not found" }
```

### 4. Update – PUT /exercises/:id
Request Body:
```json
{ "name": "Push Ups", "reps": 30, "weight": 0, "unit": "kgs", "date": "07-30-21" }
```
Success (200): updated object.  
Failure (400):
```json
{ "Error": "Invalid request" }
```
Failure (404):
```json
{ "Error": "Not found" }
```

### 5. Delete – DELETE /exercises/:id
Success (204): no body.  
Failure (404):
```json
{ "Error": "Not found" }
```

## Technical Requirements
- Only edit exercises_model.mjs and exercises_controller.mjs.  
- Use async/await with ES modules.  
- Maintain MVC separation: models handle DB logic, controllers handle HTTP.  
- Use .env for PORT (default 3000) and MONGODB_CONNECT_STRING.  
- Error responses must exactly match specification:  
  - { "Error": "Invalid request" }  
  - { "Error": "Not found" }  

## React Frontend

### Pages
1. **Home Page**  
   - Displays all exercises in a table.  
   - Delete icon → DELETE request.  
   - Edit icon → navigates to Edit Page.  
   - Link to Create Exercise Page.  

2. **Edit Exercise Page**  
   - Pre-populated form.  
   - On save → PUT request.  
   - Success/failure → alert + navigate home.  

3. **Create Exercise Page**  
   - Form with name, reps, weight, unit (select), date.  
   - On save → POST request.  
   - Success/failure → alert + navigate home.  

### Navigation
- `Navigation` component with links to Home and Create Exercise Page.  

### Styling
- Update App.css with custom colors and fonts.  
- Include header (title + description) and footer (© year, your name).  

## How to Run

### Backend
1. Unzip exercises_rest.zip.  
2. Add .env file with:  
   PORT=3000  
   MONGODB_CONNECT_STRING=<your_mongodb_connection>  
3. Install dependencies and start server:  
  ```bash
   npm install
   npm start
  ```

Backend runs at:  
  ```bash
  http://localhost:3000
  ```
### Frontend
1. Unzip exercises_react.zip.  
2. Install dependencies:  
  ```bash
   npm install
  ```
3. Start React app:  
  ```bash
   npm run dev  
  ```

Frontend runs at:  
  ```bash
  http://localhost:5173  
  ```

## Notes
- Backend must run on port 3000.  
- Frontend must run on port 5173.  
- Vite config proxies /exercises requests to backend.  
- Do not include node_modules in submission zip.
